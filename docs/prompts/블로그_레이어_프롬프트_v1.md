## 1. 삭제 (어떤 형태로도 출력 금지)
- KOSPI/KOSDAQ 예상 등락폭·트레이딩 밴드·방향 콜(상승/하락/보합)과 확신도
- 섹터 Overweight/Underweight, 비중확대/축소 등 모든 추천 표현
- "수석 분석가의 한 줄 전략" 중 매매 방향이 담긴 조언
- 특정 종목·상품에 대한 매수/매도/보유 권유로 읽힐 수 있는 모든 문장

## 2. 변환 (형태를 바꿔 보존)
- 무효화 조건 → **"시장이 주시하는 레벨"**: 방향 판단 없이 레벨만 서술.
  예) "원/달러 1,560원 돌파 시 전망 무효" → "시장 참가자들은 원/달러 1,560원 선을 주시하고 있다"
- 섹터 전략 → **"어제 실제 수급"**: 외국인·기관이 전일 실제로 순매수/순매도한 섹터·금액만 서술 (추천이 아닌 플로우 팩트)
- 지수 밴드 → **변동성 진단**으로 대체: VKOSPI 레짐(평온/경계/극단)과 최근 실현 변동폭을 서술.
  예) "VKOSPI 45로 '극단' 구간이며, 최근 5거래일 평균 일중 변동폭은 4.2%였다"
- 한 줄 전략 → **"오늘의 질문"** 1문장으로 순화 (방향 없이).
  예) "마이크론 실적이 HBM 수요 기대를 확인해줄 것인가"

## 3. 시제·화법 규칙
- 미래 추측 표현 금지: "~할 것으로 보인다", "~예상된다", "~전망이다" (본인 화법으로) 사용 불가
- 과거형·현재형 팩트 서술만 사용: "~였다", "~로 마감했다", "~를 기록 중이다"
- 제3자 인용은 출처 명시 시 허용: "OO증권은 ~로 분석했다", "시장에서는 ~라는 해석이 나온다"
- 원본 리포트에 없는 수치를 새로 만들지 않는다. 원본에 "확인 불가"인 항목은 생략하거나 "집계 시점 기준 미확인"으로 표기

## 4. 유지 (팩트이므로 그대로)
- 밤사이 미국장 지수·특징주 팩트와 그 배경 설명
- SKHY ADR 종가·환산가·괴리율 계산과 메커니즘 설명 (교육적 서술)
- 어제 한국장 마감·수급 확정치
- 오늘 이벤트 캘린더(KST 시각)
- 정책·규제 뉴스 팩트

# PART B. 구현 가이드 (Claude Code용 부록)

## B1. 아키텍처 — 이 프롬프트가 앉는 자리

```
[GitHub Actions cron: 평일 07:00 KST]
   ↓
[휴장일 체크] 한국 휴장일이면 종료
   ↓
Step 1. 리포트 생성 (LLM + 웹서치 필요)
   Claude API(web_search 도구 활성) 또는 claude -p 헤드리스에
   '한국증시_예측리포트_프롬프트_v3.md'를 실행 → 풀 리포트 생성
   ↓
[검증 게이트] 티어1 핵심 수치 8종(미 지수·선물·환율·VKOSPI·한국 지수/수급·ADR·공매도제도)
   존재 여부 파싱 체크 → 미달 시: 게시 중단, 텔레그램 오류 알림만 발송
   ↓
Step 2. 블로그 변환 (LLM 필요, 웹서치 불필요)
   본 문서 PART A를 시스템 프롬프트로, Step 1 출력을 입력으로
   → 블로그 마크다운 + 네이버 버전 생성
   ↓
Step 3. 이미지 생성 (LLM 불필요 — 순수 코드)
   전일 지수 데이터를 SVG 썸네일 템플릿에 삽입 + 지수/괴리율 차트 1~2장 (matplotlib)
   ↓
Step 4. 배포
   a) GitHub Pages 레포에 posts/YYYY-MM-DD-briefing.md + 이미지 커밋·푸시 (= 자동 게시)
   b) 텔레그램 발송: [개인용] Step 1 풀 리포트 / [복붙용] 네이버 버전 + 이미지 파일
```

## B2. 핵심 구현 노트

- **런타임 LLM이 필요한 파이프라인이다** (트레이딩 봇과 다름): Step 1·2는 Claude API 호출. Step 1이 웹서치 포함이라 비용의 대부분 (일 10~15회 검색 + 토큰). Step 2는 텍스트 변환이라 저렴. 모델: Step 1은 상위 모델, Step 2는 Sonnet급으로 충분.
- **API 키는 GitHub Secrets로 관리** (ANTHROPIC_API_KEY, TELEGRAM_BOT_TOKEN). 코드·로그에 노출 금지.
- **휴장일 캘린더**: KRX 휴장일 목록을 연 1회 갱신하는 정적 파일로 관리.
- **실패 처리**: 어느 Step이든 실패 시 게시하지 않고 텔레그램 오류 알림. "잘못된 글이 나가는 것 < 하루 거르는 것".
- **괴리율 계산 검증**: Step 2 출력의 괴리율 수치를 코드로 재계산해 대조 (SKHY×10×환율 vs 본주) — LLM 산수 오류 방지.
- **파일 규약**: posts/YYYY-MM-DD-briefing.md, 이미지 assets/YYYY-MM-DD/ 하위. frontmatter는 PART A 형식 준수 (정적 사이트 생성기가 파싱).
- **아카이브**: Step 1 풀 리포트도 비공개 폴더(reports/)에 저장 — 주간 리뷰·성과 검증 자료.

## B3. Claude Code 착수 프롬프트 (복사용)

```
이 문서(블로그_레이어_프롬프트_v1.md)의 PART B 아키텍처를 구현해줘.
전제: Astro 기반 GitHub Pages 블로그 레포(이미 있다고 가정, 없으면 먼저 스캐폴딩).
구현 순서:
1. 휴장일 체크 모듈 + GitHub Actions 워크플로우 (평일 07:00 KST cron)
2. Step 1: Anthropic API(web_search 활성)로 v3 프롬프트 실행하는 파이썬 모듈
   - 첨부: 한국증시_예측리포트_프롬프트_v3.md
3. 검증 게이트: 티어1 수치 8종 존재 파싱 + 실패 시 텔레그램 알림
4. Step 2: PART A 프롬프트로 블로그/네이버 버전 변환 + 괴리율 코드 재검증
5. Step 3: SVG 썸네일 템플릿(3대 지수 등락 자동 삽입, 상승 빨강/하락 파랑) + 괴리율 추이 차트
6. Step 4: 커밋·푸시 배포 + 텔레그램 이중 발송
- 시크릿은 전부 GitHub Secrets 참조로. 첫 실행은 dry-run 모드(게시 없이 텔레그램으로만 결과 전송)로 만들어줘.
```

- [ ] 발행 시각 (기본 07:00 KST) / dry-run 해제 시점
- [ ] 네이버 발행 루틴 (텔레그램 수신 → 복붙, 1~2분/일)
